QUICK START FOR AGENTS
═══════════════════════════════════════════════════════════════════

1. INSTALL DEPENDENCIES
   npm install ethers

2. COPY THE ABI
   Import from: bot/src/contracts/kashYieldABI.ts

3. CONNECT AND INTERACT

   import { ethers } from 'ethers';
   import { kashYieldABI } from './kashYieldABI.js';

   // Your agent's wallet
   const provider = new ethers.JsonRpcProvider('https://arb1.arbitrum.io/rpc');
   const wallet = new ethers.Wallet(process.env.AGENT_PRIVATE_KEY, provider);

   // Contract instances
   const VAULT_ADDRESS = '0x...'; // Mainnet address when deployed
   const KASH_ADDRESS = '0x...';  // KashToken address

   const vault = new ethers.Contract(VAULT_ADDRESS, kashYieldABI, wallet);
   const kashToken = new ethers.Contract(KASH_ADDRESS, ERC20_ABI, wallet);

   // DEPOSIT ETH
   const tx = await vault.depositETH({ value: ethers.parseEther('1.0') });
   await tx.wait();

   // CHECK YIELD ACCRUED (after batch processes)
   const batchCycle = await vault.getCurrentBatchCycle();
   const pending = await vault.getPendingMintRequest(wallet.address, batchCycle);

   // REDEEM
   const kashBalance = await kashToken.balanceOf(wallet.address);
   await vault.requestRedemption(kashBalance);

BATCH CYCLE EXPLANATION
═══════════════════════════════════════════════════════════════════

Deposits and redemptions are processed in daily batches at 23:50 UTC.
- Submit your request anytime
- It enters the next batch cycle
- Execution happens at 23:50 UTC
- Claim your tokens after processing completes

Why batches? Gas efficiency. All deposits/redeems processed together.
